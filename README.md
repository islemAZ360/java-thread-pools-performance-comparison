## Описание проекта

> Данный проект представляет собой практическое исследование и сравнительный анализ производительности различных пулов потоков в Java при высокой конкурентной нагрузке.

В рамках симуляции генерируется 10 000 независимых задач, каждая из которых имитирует задержку ввода-вывода (I/O) или сетевой запрос, приостанавливая выполнение на случайный промежуток времени от 100 до 500 миллисекунд. Цель теста — продемонстрировать, как различные архитектурные подходы к многопоточности справляются с мгновенным наплывом большого количества задач.

---

## Сравнительный анализ

Для оценки эффективности были протестированы три подхода. Результаты и особенности каждого из них представлены в таблице ниже.

| Тип пула | Особенности поведения при высокой нагрузке (10 000 задач) | Преимущества и недостатки |
|----------|---------------------------------------------------------|---------------------------|
| **FixedThreadPool (100 потоков)** | Ограниченное количество потоков (100). Задачи помещаются в очередь и выполняются по мере освобождения ресурсов. | Стабильное потребление памяти, отсутствие риска OutOfMemoryError. Медленное выполнение огромного пула задач из-за ожидания в очереди. |
| **CachedThreadPool** | Создает новый поток операционной системы для каждой новой задачи, если нет свободных. При 10 000 мгновенных задач может создать 10 000 OS потоков. | Идеален для множества коротких задач. При долгих I/O задачах под высокой нагрузкой может привести к исчерпанию памяти (OutOfMemoryError). |
| **Virtual Threads (Java 21)** | Использует легковесные виртуальные потоки, управляемые JVM. | Минимальное потребление памяти. Идеально подходит для I/O-bound задач. Позволяет мгновенно запустить тысячи виртуальных потоков без ущерба для ОС. |

## Как запустить

Для запуска бенчмарка убедитесь, что у вас установлена Java версии 21 или выше. Выполните следующие команды в терминале:

1. Скомпилируйте исходный код:
   ```bash
   javac -d out src/ThreadPoolBenchmark.java
   ```

2. Запустите скомпилированный класс:
   ```bash
   java -cp out ThreadPoolBenchmark
   ```

После запуска в консоли будет выведен поэтапный отчет о времени выполнения для каждого из пулов потоков, а также предупреждение, если CachedThreadPool превысит лимиты памяти операционной системы.
